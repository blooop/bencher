## Analysis

Key observations from the data above:

1. **`get_root()` + HoloViews plot rendering dominates build phase (~4.9s)**: The profiler
   shows `get_root()` takes 4.9s — constructing the Bokeh model tree, including 15
   HoloViews `get_plot()` calls at 3.8s total. This is the cost of materializing all
   HoloMap frames into Bokeh glyphs. `compute_ranges()` alone accounts for 3.4s.
2. **`embed_state()` adds 3.9s on top**: After the model is built, Panel's embed step
   replays widget state changes (60 `plot.update()` calls at 6.3s cumulative, shared
   with build). The net embed overhead is ~2.1s (baseline 4.1s minus no_embed 2.0s).
3. **`no_embed` is the floor**: Without embedding, save is just Bokeh HTML serialization.
   The gap between `no_embed` (2.0s) and `baseline` (4.1s) is pure Panel embed overhead.
4. **`max_states` / `max_opts` are TRAPS**: Setting these below the actual widget state
   count triggers Panel's `record_events` fallback, which is dramatically slower (3.1-3.3x).
   Panel computes the cross-product of all widget options; if this exceeds `max_states`,
   it switches to recording individual event replays.
5. **`inline_resources` is expensive (2.8x)**: Bundling ~4.9MB of Bokeh JS into the HTML
   happens per embed-state diff, multiplying the serialization cost.
6. **`embed_json`** does NOT skip computation — it externalizes patches to JSON files
   but still computes every diff, adding I/O overhead (1.6x slower).
7. **`max_slider_points` has minimal impact at 5-20**: With only 5 time events, the
   slider is already capped at 5 points regardless of `max_slider_points`. The scaling
   effect is only visible with more time events.
8. **Aggregated tab adds ~38% overhead**: `with_agg_tab` (5.5s) vs `no_agg_tab` (4.0s).
9. **Parallel tab saves provide NO speedup**: 0.89x (actually slower), confirming
   Panel/Bokeh is GIL-bound or has global state that prevents parallelism.

## Recommendations

Prioritized by expected impact:

1. **Keep `max_slider_points=10` (P0, shipped)** — Limits slider positions which scale
   embed cost. Effect is most visible with many time events (>10).
2. **Keep `show_aggregated_time_tab=False` (P0, shipped)** — Saves ~38% embed cost.
3. **NEVER reduce `max_states` or `max_opts` below defaults** — This is counter-intuitive
   but triggers a 3x more expensive fallback path in Panel. Keep defaults (1000/3).
4. **Optimize HoloViews `compute_ranges()` calls** — Profiling shows 3.4s in
   `compute_ranges()` across 162 calls. Caching range computation across identical
   datasets would cut the build phase significantly.
5. **Consider `embed=False` for CI/preview** — 2.1x faster (2.0s vs 4.1s), produces
   static HTML without interactive sliders. Good for automated reports.
6. **Avoid `inline_resources`** — Use CDN (default). INLINE bundles 4.9MB into the HTML,
   making it 2.8x slower.
7. **Do NOT use parallel tab saves** — Panel/Bokeh is GIL-bound; threads provide no
   speedup and add overhead. Use multiprocessing if parallelism is needed.
8. **Long-term: reduce HoloViews plot materialization cost** — The build phase
   (`get_root()`) is now comparable to embed cost. Lazy rendering or caching
   intermediate plot objects would help.
